# Firestore Realtime Monitoring Migration Requirements

## 概要
Issue #18「ポーリングによるタスク監視」を解決するため、SWRによるポーリング監視からFirestore `onSnapshot` を利用したリアルタイム監視へ移行する。

## 要件
1. **リアルタイム更新**: タスクのステータス変更（特に `pending` -> `processing` -> `completed`）が即座にUIに反映されること。
2. **セキュリティ**: ユーザーは自身のタスクのみ参照・監視できること。
3. **パフォーマンス**: 不要なポーリングリクエストを削減し、効率的なデータ取得を行うこと。
4. **一貫性**: バックエンドとフロントエンドでデータ構造（コレクション名、フィールド名）の不一致を解消すること。

## 制約事項
- Firestoreのアクセス制御ルール（Security Rules）を適切に設定すること。
- 既存のUIコンポーネントへの変更は最小限に抑えること。
